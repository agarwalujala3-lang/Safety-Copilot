# Side-by-Side Build Plan (Mentor Mode)

This is how we will work together so you become fully independent.

## Rule of this project

1. I build the real version in one folder.
2. You rebuild the simpler version in parallel.
3. We compare both and discuss why each decision was made.

## Version responsibilities

### Your simpler learning app

1. Keep only core features.
2. Keep files fewer and logic direct.
3. Focus on understanding request flow.

### Real version

1. Better structure (controller/service/store).
2. Better auth token approach.
3. Ready path to cloud and AI features.

## Learning rhythm for each module

1. Explain the feature in plain language.
2. Build with me.
3. You rebuild from memory.
4. We test.
5. We review mistakes and fix gaps.

## Modules roadmap

1. Module 1 (done): Auth + trusted circle
2. Module 2: Start trip + live location stream
3. Module 3: Arrival geofence + family notifications
4. Module 4: Risk checks (deviation, no movement, low battery, offline)
5. Module 5: SOS + silent escalation ladder
6. Module 6: AWS cloud deployment + AI risk scoring

## What to say in interviews

"I intentionally built two versions: a simple one to validate core logic and a structured version to scale for real users. This helped me iterate fast and still keep production discipline."
